## 关键概念

- `self` 不是普通实参，而是“当前这个对象自己”。在 `ReActAgent` 中，`self.history`、`self.llm_client`、`self.tool_executor` 都是这个 Agent 对象保存的状态。
- `__init__` 用于初始化 Agent，把模型客户端、工具箱、最大循环次数和历史记录保存下来。
- `response_text` 是大模型返回的完整文本，理想格式类似：
  ```text
  Thought: 我需要查询最新信息。
  Action: Search[华为最新手机]
  ```
- `_parse_output` 负责把模型输出拆成 `thought` 和 `action`。
- `_parse_action` 负责把 `Search[关键词]` 拆成工具名和工具输入。
- `Finish[...]` 表示任务结束并返回最终答案；`Search[...]` 表示需要继续调用工具。
- 工具调用不是由 GPT 直接执行，而是由 Python 根据工具名找到真实函数并调用。

## 今天读懂的代码

- `ReActAgent.run`
  - 每一轮构造 prompt。
  - 调用 `llm_client.think(messages=messages)` 获取模型输出。
  - 用 `_parse_output(response_text)` 提取 `Thought` 和 `Action`。
  - 如果 `Action` 是 `Finish[...]`，直接返回最终答案。
  - 如果 `Action` 是工具调用，就解析工具名和输入。
  - 从工具箱中找到对应函数并执行。
  - 将 `Action` 和 `Observation` 写入 `history`，供下一轮模型推理使用。

- `ToolExecutor`
  - `registerTool` 把工具名、描述和真实函数保存到工具箱。
  - `getTool` 根据工具名取出真实 Python 函数。
  - `getAvailableTools` 把工具描述整理成 prompt 中可读的工具列表。
