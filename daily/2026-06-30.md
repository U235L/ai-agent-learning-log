## 关键概念

- `FakeLLM` 是教学用的假大模型，用固定规则模拟真实 LLM 的输出。
- 在教学版中，用是否出现 `Observation` 简化判断；但真实 Agent 会根据完整上下文决定继续调用工具还是结束。
- 工具调用的本质是：用字典把“工具名字符串”映射到“真实 Python 函数”。
- `func=fake_search` 表示保存函数本身；`fake_search(...)` 才表示立即调用函数。
- `Callable[[str], str]` 表示一个可调用对象，接收一个字符串参数并返回一个字符串。
- Action 格式错误和工具不存在是两种不同错误：
  - 格式错误：无法解析出工具名和输入。
  - 工具不存在：格式正确，但工具箱没有注册对应工具。

## 今日重点理解

模型输出：

```text
Action: Search[AI Agent 是什么]
```

代码解析出：

```python
tool_name = "Search"
tool_input = "AI Agent 是什么"
```

工具箱查找：

```python
tool_function = self.tool_executor.get_tool("Search")
```

得到真实函数：

```python
tool_function = fake_search
```

最终执行：

```python
observation = fake_search("AI Agent 是什么")
```

这个过程说明：GPT/LLM 并不是自己执行工具，而是输出动作指令；Python 负责解析、验证、执行工具并记录结果。
